Technical Blog Part 1

This week we got to learn more about CSS frameworks and responsive design.

Grid based design

An effective way for creating a webpage is to use a 12-column grid. Each row is divided into 12 columns of equal width. Each element is assigned a class that represents the total number of columns an element can spans across a page. For example, if I assign an element a class of "col-6", the element will use 50% of the pages' width, regardless of which device i'm using. We use 12 columns because it is easily divisible by 2, 3, 4, 6, and 12.

Responsive web design

People use many different devices to access the internet. Every one of these devices uses a different size screen and therefore has a different resolution. To ensure that a webpage looks good from any device, content needs to adjust dynamically according to the devices' resolution. Following the principals of Responsive design, webpage content is not fixed to a certain size.

Semantic Tags

Once you have figured out the structure of your page, you can start to think about what content is most important and should stand out. Once you have identified the priority of your elements you can use Semantic tags to identify them. For example:

  • Header 1 - h1: For more prominent headings such as a page title
  • Header 2 - h2: For less prominent headings such as a section title
  • Header 3 - h3: For articles
